# AevumDB Security Policy

## Supported Versions
We actively provide security updates for the following versions of AevumDB:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | ✅ Yes             |
| < 1.0   | ❌ No              |

## Reporting a Vulnerability
We appreciate the hard work of security researchers in keeping AevumDB safe. If you discover a security vulnerability, please **DO NOT** create a public Issue. Instead, follow ini these steps:

1. Email the details of the vulnerability to [aevumdb@gmail.com](mailto:aevumdb@gmail.com).
2. Include a technical description, steps to reproduce (PoC), and the potential impact.
3. Our team will acknowledge receipt of your report within 48 hours.

## Disclosure Process
Once a report is received:
1. We will work with you to validate and fix the vulnerability.
2. We ask that you maintain confidentiality regarding the issue until we have released an official fix (**Responsible Disclosure**).
3. Once the fix is released, we will give credit to you in our release notes unless you prefer to remain anonymous.

## Scope
This policy applies to:
* AevumDB Kernel (C++/Rust).
* Official Network Protocols and APIs.
* Official client drivers maintained under the AevumDB organization.

Thank you for helping keep the AevumDB ecosystem secure!
